# AGENTS.md

## Cursor Cloud specific instructions

This is a Julia package (RestrictedBoltzmannMachines.jl). Julia is installed via `juliaup` at `~/.juliaup/bin` — ensure `PATH` includes it (the update script handles this).

### Quick reference

Commands for common tasks are documented in `CLAUDE.md` under "Common Commands". Key commands:

- **Run all tests:** `julia --project=. -e 'using Pkg; Pkg.test()'`
- **Run a single test file:** `julia --project=test test/<file>.jl`
- **Load the package:** `julia --project=. -e 'import RestrictedBoltzmannMachines as RBMs'`

### Non-obvious caveats

- The project uses a Julia workspace (`[workspace]` in root `Project.toml`). A single shared `Manifest.toml` at the repo root covers all sub-projects (`test`, `docs`, `notebooks`, `repl`). Always run `--project=.` for the main package and `--project=test` for tests.
- The test sub-project must have the root package dev'd into it. The update script runs `Pkg.develop(PackageSpec(path=pwd()))` inside `--project=test` to set this up.
- `Manifest.toml` is gitignored and generated by `Pkg.instantiate()`. If you see resolution errors, delete it and re-instantiate.
- No external services (databases, Docker, etc.) are required. The package is a pure Julia library.
- CUDA extension (`ext/CUDAExt.jl`) is optional and only loads when `CUDA.jl` is available (requires an NVIDIA GPU). Cloud VMs typically lack a GPU — this is expected and does not affect CPU tests.
- HDF5 extension (`ext/HDF5Ext.jl`) binary dependency is handled automatically by Julia's artifact system.
- Test precompilation on first run after instantiation can take ~2 minutes. Subsequent runs are much faster.
